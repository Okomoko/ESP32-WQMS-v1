Dashboard Automation Configuration Calibration API Docs

API Documentation

All endpoints return JSON unless otherwise noted. Base URL: http://192.168.1.100

System

GET /api/status

Get system status including name, version, IP, MAC, WiFi, uptime, memory, NTP and MODBUS status.

Example Response:
{
    "system_name": "WQMS-PlantLab",
    "version": "v1.0.0",
    "ip": "192.168.1.100",
    "mac": "A1:B2:C3:D4:E5:F6",
    "wifi_ssid": "PlantLab",
    "wifi_mode": "STA",
    "uptime": "2d 3h 45m",
    "uptime_seconds": 186300,
    "free_heap_bytes": 234567,
    "free_heap_kb": 229,
    "cpu_temp_c": 45.2,
    "ntp_synced": true,
    "timezone": "EET-3",
    "modbus_packets": 1234,
    "modbus_errors": 0
}
cURL:
curl -X GET http://192.168.1.100/api/status

GET /api/echo

Simple health check endpoint. Returns a JSON object with status and response.

Example Response:
{
    "status": "OK",
    "code": 200,
    "response": "WQMS"
}
cURL:
curl -X GET http://192.168.1.100/api/echo

POST /api/system/reboot

Reboot the system.

Example Response:
{"success": true, "message": "System rebooting..."}
cURL:
curl -X POST http://192.168.1.100/api/system/reboot

Sensors

GET /api/sensors

Get all current sensor readings.

Example Response:
{
    "sensors": [
        {"id": 0, "name": "pH", "value": 7.23, "raw_adc": 2345, "status": 0, "quality": 0, "timestamp": 1718622225},
        {"id": 1, "name": "EC", "value": 1.45, "raw_adc": 1567, "status": 0, "quality": 0, "timestamp": 1718622225}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/sensors

GET /api/sensors/config

Get sensor configuration including names, GPIO pins, MODBUS registers, min/max values, and calibration factors.

Example Response:
{
    "sensors": [
        {"id": 0, "name": "pH", "enabled": true, "gpio_pin": 32, "modbus_register": 0, "min_value": 0, "max_value": 14, "calibration_factor": 1000}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/sensors/config

POST /api/sensors/config

Update sensor configuration (names, enabled status, calibration factors).

Request Body:
{
    "sensors": [
        {"id": 0, "name": "pH Sensor", "enabled": true, "calibration_factor": 1234}
    ]
}
Example Response:
{"success": true, "message": "Sensor configuration saved"}
cURL:
curl -X POST http://192.168.1.100/api/sensors/config -d '{"sensors":[{"id":0,"name":"pH Sensor","enabled":true}]}' -H "Content-Type: application/json"

Relays

GET /api/relays

Get all relay status.

Example Response:
{
    "relays": [
        {"id": 0, "name": "Pump 1", "state": 0, "active": false, "remaining": 0, "enabled": true, "duration": 100, "off_delay": 50, "gpio_pin": 25}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/relays

POST /api/relays/{id}/trigger

Trigger a relay (ON for configured duration). Optionally specify custom duration in milliseconds.

Request Body:
{"duration": 500}
Example Response:
{"success": true, "message": "Relay triggered"}
cURL:
curl -X POST http://192.168.1.100/api/relays/0/trigger -d '{"duration":500}' -H "Content-Type: application/json"

POST /api/relays/{id}/off

Manually turn off a relay.

Example Response:
{"success": true, "message": "Relay turned off"}
cURL:
curl -X POST http://192.168.1.100/api/relays/0/off

GET /api/relays/config

Get relay configuration including names, GPIO pins, MODBUS registers, durations, and off-delays.

Example Response:
{
    "relays": [
        {"id": 0, "name": "Pump 1", "enabled": true, "gpio_pin": 25, "modbus_register": 256, "duration_ms": 100, "off_delay_ms": 50}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/relays/config

POST /api/relays/config

Update relay configuration (names, enabled status, durations, off-delays).

Request Body:
{
    "relays": [
        {"id": 0, "name": "Pump 1", "enabled": true, "duration_ms": 200, "off_delay_ms": 100}
    ]
}
Example Response:
{"success": true, "message": "Relay configuration saved"}
cURL:
curl -X POST http://192.168.1.100/api/relays/config -d '{"relays":[{"id":0,"name":"Pump 1","duration_ms":200}]}' -H "Content-Type: application/json"

WiFi

GET /api/wifi/status

Get current WiFi status including mode, SSID, IP, MAC, connection state, and RSSI.

Example Response:
{
    "mode": "STA",
    "mode_id": 1,
    "ssid": "PlantLab",
    "ip": "192.168.1.100",
    "mac": "A1:B2:C3:D4:E5:F6",
    "connected": true,
    "rssi": -45
}
cURL:
curl -X GET http://192.168.1.100/api/wifi/status

GET /api/wifi/scan

Scan for available WiFi networks.

Example Response:
{
    "networks": [
        {"ssid": "PlantLab", "rssi": -45, "secured": true},
        {"ssid": "Office", "rssi": -68, "secured": true}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/wifi/scan

POST /api/wifi/connect

Connect to a WiFi network. Credentials are only saved on success. Also supports setting mode.

Request Body (Connect to STA):
{"ssid": "PlantLab", "password": "mysecret"}
Request Body (Set Mode):
{"mode": 1}
Example Response (Success):
{"success": true, "message": "Connected to network! Reconnect to new IP.", "ssid": "PlantLab", "ip": "192.168.1.100"}
cURL:
curl -X POST http://192.168.1.100/api/wifi/connect -d '{"ssid":"PlantLab","password":"mysecret"}' -H "Content-Type: application/json"

POST /api/wifi/forget

Forget the current WiFi network. System will reboot into AP mode.

Example Response:
{"success": true, "message": "Network forgotten, system will restart"}
cURL:
curl -X POST http://192.168.1.100/api/wifi/forget

Calibration

POST /api/calibrate/start

Start a calibration session for a sensor.

Request Body:
{"sensor_id": 0}
Example Response:
{"success": true, "message": "Calibration started", "session_id": "cal_20260617_102345"}
cURL:
curl -X POST http://192.168.1.100/api/calibrate/start -d '{"sensor_id":0}' -H "Content-Type: application/json"

POST /api/calibrate/sample

Add a reference sample to the current calibration session.

Request Body:
{"known_value": 4.01}
Example Response:
{"success": true, "sample_number": 1, "raw_adc": 2345, "voltage": 1.89, "known_value": 4.01}
cURL:
curl -X POST http://192.168.1.100/api/calibrate/sample -d '{"known_value":4.01}' -H "Content-Type: application/json"

POST /api/calibrate/apply

Apply the calibration factor from the current session.

Example Response:
{"success": true, "factor": 1.234, "method": "linear_regression", "confidence": 0.952}
cURL:
curl -X POST http://192.168.1.100/api/calibrate/apply

POST /api/calibrate/cancel

Cancel the current calibration session.

Example Response:
{"success": true, "message": "Calibration cancelled"}
cURL:
curl -X POST http://192.168.1.100/api/calibrate/cancel

GET /api/calibrate/factor

Get the current calibration factor preview.

Example Response:
{"success": true, "factor": 1.234}
cURL:
curl -X GET http://192.168.1.100/api/calibrate/factor

Logging

GET /api/logs

List all log files.

Example Response:
{
    "logs": [
        {"name": "system_0.log", "size": 234567, "modified": 1718622225}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/logs

GET /api/logs?name={filename}

Download a specific log file.

cURL:
curl -X GET http://192.168.1.100/api/logs?name=system_0.log --output system_0.log

DELETE /api/logs?name={filename}

Delete a specific log file.

cURL:
curl -X DELETE http://192.168.1.100/api/logs?name=system_0.log

DELETE /api/logs/all

Delete all log files.

cURL:
curl -X DELETE http://192.168.1.100/api/logs/all
