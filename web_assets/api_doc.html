API Documentation

All endpoints return JSON unless otherwise noted. Base URL: http://192.168.1.100

System

GET /api/status

Get system status including name, version, IP, MAC, WiFi, uptime, memory, NTP and MODBUS status.

Example Response:
{
    "system_name": "WQMS-PlantLab",
    "version": "v1.0.0",
    "ip": "192.168.1.100",
    "mac": "A1:B2:C3:D4:E5:F6",
    "wifi_ssid": "PlantLab",
    "wifi_mode": "STA",
    "uptime": "2d 3h 45m",
    "uptime_seconds": 186300,
    "free_heap_bytes": 234567,
    "free_heap_kb": 229,
    "cpu0_load": 12,
    "cpu1_load": 8,
    "ntp_synced": true,
    "timezone": "EET-3",
    "modbus_packets": 1234,
    "modbus_errors": 0
}
cURL:
curl -X GET http://192.168.1.100/api/status

GET /api/echo

Simple health check endpoint. Returns a JSON object with status and response.

Example Response:
{
    "status": "OK",
    "code": 200,
    "response": "WQMS"
}
cURL:
curl -X GET http://192.168.1.100/api/echo

POST /api/system/reboot

Reboot the system.

Example Response:
{"success": true, "message": "System rebooting..."}
cURL:
curl -X POST http://192.168.1.100/api/system/reboot

Sensors

GET /api/sensors

Get all current sensor readings.

Example Response:
{
    "sensors": [
        {"id": 0, "name": "pH", "value": 7.23, "raw_adc": 2345, "status": 0, "quality": 0, "timestamp": 1718622225},
        {"id": 1, "name": "EC", "value": 1.45, "raw_adc": 1567, "status": 0, "quality": 0, "timestamp": 1718622225}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/sensors

GET /api/sensors/config

Get sensor configuration including names, GPIO pins, MODBUS registers, min/max values, units, and calibration factors.

Example Response:
{
    "sensors": [
        {"id": 0, "name": "pH", "enabled": true, "unit": 2, "gpio_pin": 32, "adc_channel": 0, "modbus_register": 0, "min_value": 0, "max_value": 14, "calibration_factor": 1000, "current_value": 7.23, "status": 0}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/sensors/config

POST /api/sensors/config

Update sensor configuration (names, enabled status, calibration factors, units).

Request Body:
{
    "sensors": [
        {"id": 0, "name": "pH Sensor", "enabled": true, "calibration_factor": 1234, "unit": 2}
    ]
}
Example Response:
{"success": true, "message": "Sensor configuration saved"}
cURL:
curl -X POST http://192.168.1.100/api/sensors/config -d '{"sensors":[{"id":0,"name":"pH Sensor","enabled":true}]}' -H "Content-Type: application/json"

WiFi

GET /api/wifi/status

Get current WiFi status including mode, SSID, AP SSID, IP, MAC, connection state, and RSSI.

Example Response:
{
    "mode": "STA",
    "mode_id": 1,
    "ssid": "PlantLab",
    "ap_ssid": "WQMS-XXXX",
    "ap_password": "WQMSwqms",
    "ip": "192.168.1.100",
    "mac": "A1:B2:C3:D4:E5:F6",
    "connected": true,
    "rssi": -45
}
cURL:
curl -X GET http://192.168.1.100/api/wifi/status

GET /api/wifi/scan

Scan for available WiFi networks. Returns networks with SSID, RSSI, security info, and channel.

Example Response:
{
    "networks": [
        {"ssid": "PlantLab", "rssi": -45, "channel": 6, "secured": true, "authmode": "WPA2"},
        {"ssid": "Office", "rssi": -68, "channel": 1, "secured": true, "authmode": "WPA2"}
    ],
    "total": 2
}
cURL:
curl -X GET http://192.168.1.100/api/wifi/scan

POST /api/wifi/connect

Connect to a WiFi network. Credentials are only saved on success. Also supports setting mode.

Request Body (Connect to STA):
{"ssid": "PlantLab", "password": "mysecret"}
Request Body (Set Mode):
{"mode": 1}

Mode values: 0=AP, 1=STA, 2=AP+STA

Example Response (STA Success):
{"success": true, "message": "Connected to network! Reconnect to new IP.", "ssid": "PlantLab", "ip": "192.168.1.100"}
Example Response (AP Mode):
{"success": true, "message": "Switched to AP mode. Reconnect to new network.", "ssid": "WQMS-XXXX", "password": "WQMSwqms", "ip": "192.168.4.1"}
cURL:
curl -X POST http://192.168.1.100/api/wifi/connect -d '{"ssid":"PlantLab","password":"mysecret"}' -H "Content-Type: application/json"

POST /api/wifi/forget

Forget the current WiFi network. System will reboot into AP mode.

Example Response:
{"success": true, "message": "Network credentials are cleared, system will restart"}
cURL:
curl -X POST http://192.168.1.100/api/wifi/forget

Calibration

POST /api/calibrate/start

Start a calibration session for a sensor.

Request Body:
{"sensor_id": 0}
Example Response:
{"success": true, "message": "Calibration started", "sensor_id": 0}
cURL:
curl -X POST http://192.168.1.100/api/calibrate/start -d '{"sensor_id":0}' -H "Content-Type: application/json"

POST /api/calibrate/sample

Add a reference sample to the current calibration session.

Request Body:
{"known_value": 4.01}
Example Response:
{"success": true, "sample_number": 1, "raw_adc": 2345, "voltage": 1.89, "known_value": 4.01, "factor": 1.234}
cURL:
curl -X POST http://192.168.1.100/api/calibrate/sample -d '{"known_value":4.01}' -H "Content-Type: application/json"

POST /api/calibrate/apply

Apply the calibration factor from the current session. Requires at least 2 samples.

Example Response:
{"success": true, "factor": 1.234, "message": "Calibration applied successfully"}
cURL:
curl -X POST http://192.168.1.100/api/calibrate/apply

POST /api/calibrate/cancel

Cancel the current calibration session.

Example Response:
{"success": true, "message": "Calibration cancelled"}
cURL:
curl -X POST http://192.168.1.100/api/calibrate/cancel

GET /api/calibrate/factor

Get the current calibration factor preview and sample count.

Example Response:
{"success": true, "factor": 1.234, "sample_count": 3, "message": "Factor calculated"}
cURL:
curl -X GET http://192.168.1.100/api/calibrate/factor

Logging

GET /api/logs

List all log files. Includes the circular buffer system log and any legacy log files.

Example Response:
{
    "logs": [
        {"name": "system.log", "size": 234567, "modified": 1718622225},
        {"name": "old_log_1.log", "size": 12345, "modified": 1718622000}
    ]
}
cURL:
curl -X GET http://192.168.1.100/api/logs

GET /api/logs?name={filename}

Download a specific log file. For system.log, streams from the circular buffer.

cURL:
curl -X GET http://192.168.1.100/api/logs?name=system.log --output system.log

DELETE /api/logs?name={filename}

Delete a specific log file. For system.log, clears the circular buffer.

cURL:
curl -X DELETE http://192.168.1.100/api/logs?name=system.log

DELETE /api/logs/all

Delete all log files including clearing the circular buffer.

cURL:
curl -X DELETE http://192.168.1.100/api/logs/all

GET /api/history

Get historical sensor data. Returns up to 4320 records (3 days at 1-minute intervals).

Query Parameters:

Example Response:
{
    "entries": [
        {"timestamp": 1718622225, "sensor_mask": 255, "pH": 7.23, "EC": 1.45},
        {"timestamp": 1718622285, "sensor_mask": 255, "pH": 7.25, "EC": 1.43}
    ]
}
cURL:
curl -X GET "http://192.168.1.100/api/history?limit=60"

GET /api/history/config

Get sensor configuration for history chart (names, IDs, colors).

Example Response:
{
    "sensors": [
        {"id": 0, "name": "pH", "color": "#00FFFF"},
        {"id": 1, "name": "EC", "color": "#FF00FF"}

}
cURL:
curl -X GET http://192.168.1.100/api/history/config

GET /api/history/stats
